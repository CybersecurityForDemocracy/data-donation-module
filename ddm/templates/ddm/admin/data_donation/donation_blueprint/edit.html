Data Extraction Settings

Data Extraction is a two-step process consisting of first the file validation and second the data extraction.

File Validation
  • First, it is checked whether the expected file is included in the uploaded data (only applies to ZIP uploads). If the associated File Uploader expects a ZIP Upload, the correct file is identified using the provided file path (this is skipped for single file uploads).
  • Second, it is checked whether the uploaded file is in the expected file format.
  • Third, it is checked whether the identified file contains all expected fields.
  • If any of these validation steps fail, the participant will be shown an exception message explaining what went wrong and no data is extracted.

Data Extraction
  • The data extraction follows extraction rules which can be configured below. These rules are applied consecutively in the defined order.
  • Keep data: For every field/column/variable that you want to keep in the donated data, you first have to define an extraction rule with the "Keep field" operator.
  • Filter and alter data: Next, you can add rules to filter (i.e., delete) or alter entries in the uploaded data (e.g., to delete all entries where the date is < 01.01.2020, or to replace e-mail-addresses with "ANONYMIZED EMAIL"). For this, there are several comparison and regex operations available.
